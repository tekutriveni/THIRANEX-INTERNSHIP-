## 📁 Project — FoodRush Food Delivery App

> A complete UI/UX design project for **FoodRush** — a food delivery platform. From low-fidelity mobile wireframes to high-fidelity website redesign to a fully interactive clickable prototype.

---

## ✅ Week 1 — Mobile App Wireframing

### 🎯 Objective
Design low-fidelity wireframes for a mobile food delivery application.

### 🔍 Key Deliverables
- ✅ User Research & Pain Points Analysis
- ✅ User Personas (Priya & Arun)
- ✅ User Flow Mapping
- ✅ Low-Fidelity Wireframes (5 screens)
- ✅ Design Thinking Documentation

### 📱 Screens Designed
| Screen | Description |
|--------|-------------|
| Splash / Onboarding | First impression, branding, Get Started CTA |
| Home Screen | Browse restaurants, featured items, search |
| Restaurant Menu | Food items, categories, add to cart |
| Cart & Checkout | Review order, apply offers, payment |
| Order Confirmation | Success screen, ETA, tracking |

## ✅ Week 2 — Website UI Redesign

### 🎯 Objective
Redesign the FoodRush website UI with high-fidelity mockups, style guide, and responsive layouts.

### 🔍 Key Deliverables
- ✅ Heuristic Evaluation (8 usability issues identified)
- ✅ Mood Board & Visual Direction
- ✅ Complete Style Guide (Colors, Typography, Components)
- ✅ High-Fidelity Mockups (4 pages)
- ✅ Responsive Layout Strategy (Desktop / Tablet / Mobile)

### 🖥️ Pages Redesigned
| Page | Key Features |
|------|-------------|
| Home Page | Hero banner, category filters, restaurant grid |
| Restaurant Page | Sticky tabs, menu items, quick-add, reviews |
| Cart & Checkout | 3-step progress, address form, payment options |
| Order Tracking | Live map, driver info, progress timeline, ETA |

### 🎨 Design System
- **Primary Color** — `#FF6B21` Orange
- **Dark** — `#0F0F0F` Black
- **Success** — `#2EB872` Green
- **Headings** — Syne (700, 800)
- **Body** — DM Sans (300, 400, 500, 600)

## ✅ Week 3 — Interactive Prototype Creation

### 🎯 Objective
Build a fully clickable interactive prototype to simulate real app interactions and conduct quick user testing.

### 🔍 Key Deliverables
- ✅ 8 Screens connected with Click interactions
- ✅ Dissolve transitions between all screens
- ✅ Complete user journey — Splash to Order Tracking
- ✅ User Testing (6 scenarios, 3 participants)
- ✅ Design Refinements based on feedback

### 📱 Prototype Flow
```
01 Splash → 02 Onboarding → 03 Home → 04 Restaurant → 05 Cart → 06 Checkout → 07 Confirmed → 08 Tracking
```

### 🖱️ Interactions Used
| Interaction | Details |
|-------------|---------|
| Trigger | On Click |
| Action | Navigate To |
| Animation | Dissolve |
| Frame Size | 390×844px (iOS/Android) |
| Starting Point | 01 Splash Screen (Flow 1) |

### 🧪 User Testing Results
| Scenario | Result |
|----------|--------|
| Onboarding Flow | ✅ Pass |
| Restaurant Discovery | ✅ Pass |
| Add to Cart | ✅ Pass |
| Checkout Flow | ⚡ Needs Improvement |
| Order Confirmation | ✅ Pass |
| Overall Navigation | ⭐ 4.2 / 5 |
